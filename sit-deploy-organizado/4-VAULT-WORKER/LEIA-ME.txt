VAULT WORKER — INSTRUÇÕES (OPCIONAL)

Este passo só é necessário se você já tem um Vault Worker 
rodando (The Arch / Cardinal) e quer conectar o SIT a ele
para análise pelos conselheiros (Kahneman, Taleb, Buffett etc).

1. Abra seu vault-worker.js existente no Cloudflare Dashboard
2. Copie as funções de vault-worker-sit-patch.js
3. Cole DENTRO do seu worker, antes do "return json({error:'Not found'},404)"
4. Adicione no router principal:
     if (path === '/sit/analyze') return await sitAnalyze(request, env);
     if (path === '/sit/news')    return await sitNews(request, env);
     if (path === '/sit/history') return await sitHistory(url, env);
5. Adicione as variáveis de ambiente no Worker:
     GNEWS_API_KEY = sua key de gnews.io
     (ANTHROPIC_KEY já deve existir se o Vault já funciona)
6. Deploy do worker

Se você NÃO tem Vault ainda, pule esta etapa.
O SIT funciona perfeitamente sem o Vault — só não terá
a análise dos conselheiros.
